# 申请表信息字段

字段按官方申请表结构分组。

## 软件申请信息

| 序号 | 字段 | 说明 |
|---|---|---|
| 1 | 软件全称 | 必须由用户确认；最终正式资料文件名、代码页眉和操作手册中的软件名称均以此为准 |
| 2 | 软件简称 | 非必填，默认"（无）" |
| 3 | 版本号 | 必须由用户确认；若项目版本 < V1.0，提醒用户软著首次提交通常写 V1.0 |
| 4 | 著作权人 | 待用户确认 |
| 5 | 著作权人类型 | 默认"企业法人" |
| 6 | 权利范围 | 默认"全部权利" |

## 软件开发信息

| 序号 | 字段 | 说明 |
|---|---|---|
| 7 | 软件分类 | 四选一：应用软件 / 嵌入式软件 / 中间件 / 操作系统，默认"应用软件" |
| 8 | 软件说明 | 默认"原创" |
| 9 | 开发方式 | 默认"单独开发" |
| 10 | 开发完成日期 | 待用户确认 |
| 11 | 首次发表日期 | 默认"未发表（首次发表）" |

## 字段口径说明

- 软件全称：必须由用户确认；最终正式资料文件名、代码页眉和操作手册中的软件名称均以本字段为准。
- 软件开发环境 / 开发工具：填写 IDE 或编辑器名称，例如 Visual Studio Code、WebStorm、IntelliJ IDEA、Cursor。
- 版本号：必须由用户确认；如果项目版本小于 V1.0，软著首次提交通常建议使用 V1.0，也可按实际项目版本填写。
- 开发该软件的操作系统：填写实际开发电脑的操作系统版本，例如 macOS 14、macOS 15、Windows 10、Windows 11。
- 该软件的运行平台 / 操作系统：填写软件客户端或服务运行所在的操作系统版本。
- 软件运行支撑环境 / 支持软件：填写项目运行所需的软件环境，例如 Node.js、Python、Docker、数据库、浏览器、中间件或外部服务。
- 开发的硬件环境和运行的硬件环境：可使用当前检测到的电脑配置作为建议值，也可按实际开发、部署或审核口径调整。

## 字数限制规则

以下字段在正式输出前必须校验字数（去空格和换行后计算）：

| 字段 | 限制 | 校验方式 |
|---|---|---|
| 开发的硬件环境、运行的硬件环境、开发该软件的操作系统、软件开发环境 / 开发工具、该软件的运行平台 / 操作系统、软件运行支撑环境 / 支持软件 | ≤50 字 | 超限则精简缩写或省略次要组件 |
| 开发目的、面向领域 / 行业 | ≤50 字 | 超限则缩句，保留核心语义 |
| 编程语言 | ≤120 字 | 超限则摘取项目实际使用的语言，省略未使用的 |
| 软件的主要功能 | 500-1300 字 | 不足则补写架构/场景/价值链条，超限则精简冗余描述 |

校验逻辑在 `generate_application_info.py` 的 `write_application_md()` 中实现：字数不达标时输出 `FIELD_WARNING` 警告并在 STOP_FOR_USER 块中提示。

## 一致性要求

- 软件全称和版本号必须与代码材料、操作手册一致。
- 正式代码 Word 页眉软件名称必须与申请表信息中的"软件全称"一致。
- 正式代码 Word 页眉版本号必须与申请表信息中的"版本号"一致。
- 主要功能必须来自当前项目，不得沿用范本中的旧项目描述。
- 操作手册截图中的软件名称、公司 Logo 必须与申请表信息保持绝对一致。
- `待用户确认` 字段在正式输出前必须替换为确认值；未补全时不得记录 `application-fields` 门禁。
