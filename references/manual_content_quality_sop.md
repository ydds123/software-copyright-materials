# 操作手册内容质量 SOP

操作手册中每个功能模块的撰写深度不应一刀切。有些模块是纯台账（CRUD），有些模块蕴含复杂的业务逻辑。用同样的四表骨架写两者会导致业务型功能被严重削弱——比如审批备案被写成"点击发起审批，系统推送任务"，而实际上它涉及数据快照、BPMN 签批流转、新旧数据 Diff 对比、十条业务模块的并行变更记录、审批通过后的快照覆盖和公司类型升级。

核心原则：**识别模块类型 → 按类型决定撰写深度 → 非技术语言 + 表格化呈现 → 功能清单迭代升级**。

## 一、模块三分类

写完功能清单后，逐模块判定类型：

| 类型 | 判断标准 | 示例 |
|---|---|---|
| **台账型** | 页面只有列表查询+新增/修改/删除/导入/导出，没有审批流转、状态变迁或多角色协同 | 设备管理、二维码管理、NFC 管理 |
| **业务型** | 页面涉及审批流程、多角色操作、状态机流转、数据版本管理，或代码中有 Controller/Service/Event 等多层后端逻辑 | 承包商审批备案、巡检任务执行、隐患治理 |
| **混合型** | 既有台账面（列表查询+CRUD），又有业务配置面（如关联审批规则、双重预防开关等） | 承包商信息配置、巡检点管理、巡检计划管理 |

## 二、按类型决定撰写深度

### 台账型

- 列表界面表：逐列列出所有列名（不准用"等"字），声明排序和分页
- 新增/修改界面字段表：6 列表格（名称/类型/必填/默认/边界/规则）
- 异常功能逻辑表：至少 4 条
- 在功能概要中注明该台账的数据被哪个下游业务模块使用——例如"设备管理维护的设备台账被巡检点管理引用为检查对象"
- **不需要深挖后端逻辑**——台账型功能的代码链路就是 Controller→Service→Mapper，没有值得展开的业务细节

### 业务型

- 列表界面表：标准 8 行
- **操作步骤或状态流转表**（代替单纯的字段表）：如果该功能有审批流或多步骤操作，用"步骤/用户操作/系统响应"三列表格呈现全链路
- 异常功能逻辑表：至少 4 条，覆盖每个关键步骤的异常场景
- **必须深挖后端代码**：找到 Controller 端点、Service 实现、事件监听器、状态枚举常量。弄清楚：
  - 操作触发后系统做了什么（数据快照？启动流程？发布事件？）
  - 状态在哪里流转、有哪些状态值、触发条件是什么
  - 涉及多少业务模块参与——像审批有十个并行模块
  - 失败时如何回退、回退到哪里
- **技术术语全部翻译为业务语言**：不准出现"Flowable""BPMN""Spring Event""CountDownLatch"。写成"审批流程""系统自动将任务推送给下一签批人"
- **字不如表，表不如图**：
  - 一段描述 → 拆成操作步骤表
  - 状态说明 → 用"状态/含义/后果"三列表
  - 多角色交互 → 用"角色/操作/系统响应"三列表
  - 差异对比 → 用颜色标记表（绿色新增、红色删除）

### 混合型

- 台账面按台账型处理，业务面按业务型处理
- 如果配置面有独立子页面（如"审批流程配置"），拆为独立的子节处理
- 必须在台账面注明其配置如何影响下游业务模块的行为

## 三、功能清单迭代

功能清单不是一次写完的。在完成各模块详细描述后，回到功能清单表，用深挖到的细节升级：

- 台账型字段保持不变
- 业务型字段补上流程关键环节——例如"发起审批与跟踪"不能只写这六个字，要展开为"发起审批（数据快照+十模块差异记录+流程启动）、签批审核（新旧数据对比+提交/转办/终止）、审批通过（快照覆盖+公司类型升级+进入名录库）、审批终止（状态回退+草稿恢复）"

## 四、自查清单

在输出 STOP_FOR_USER 前，逐条确认：

| # | 检查项 | 判定 |
|---|---|---|
| 1 | 是否识别了所有业务型模块？ | 每个模块确认类型，写在模块标题后 |
| 2 | 业务型模块是否读了后端 Controller/Service/Event 代码？ | 阅读代码路径，提取完整业务链路 |
| 3 | 是否所有技术术语都被替换为业务语言？ | 搜索"Event""BPMN""Listener""Flowable""流程定义键"等词——全部清除 |
| 4 | 是否做到了"字不如表、表不如图"？ | 超过 3 句连续描述的段落必须改为表格或流程图 |
| 5 | 操作步骤是否用三列表（步骤/操作/响应）呈现？ | 纯 CRUD 模块可以只用字段表；有流程的模块必须有操作步骤表 |
| 6 | 审批/流转类功能的异常逻辑是否覆盖了每个关键步骤？ | 不仅是"必填为空"和"网络中断"，还要覆盖"审批中退回""终止后草稿恢复""节点转办"等 |
| 7 | 功能清单是否在模块细节写完后进行了迭代升级？ | 对比初版和终版的功能清单，确认业务型模块的子功能描述已展开 |
